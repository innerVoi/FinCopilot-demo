# FinCopilot V2.2 完整 Demo 规划

## 1. 版本目标

FinCopilot V2.2 的目标是将 V2.1 的多 Agent 能力收敛到一个统一 Copilot 主界面中。

用户可以在主界面完成：

- 上传数据；
- 查看数据状态；
- 选择推荐任务；
- 自然语言提问；
- 查看结果卡片；
- 展开详细结果预览；
- 生成行动项；
- 查看报告和 Trace。

## 2. 演示主线

1. 打开 Copilot 主界面；
2. 展示无数据状态；
3. 使用默认样例数据；
4. 点击推荐任务“检查未来 30 天现金流”；
5. 展示结果卡片；
6. 展示详细结果预览；
7. 展示分析详情；
8. 展示行动与报告；
9. 展示数据与设置；
10. 展示 fallback 机制；
11. 总结安全边界。

## 3. 核心卖点

- 一个主界面完成主要任务；
- 默认尝试真实 Agent；
- 失败自动 fallback；
- 多 Agent 分析结果卡片化；
- 详细结果可在主界面展开；
- 详情页保留完整证据；
- 新手引导降低使用门槛；
- 不提供高风险专业建议。

## 4. fallback 展示

无 `OPENAI_API_KEY`、模型不可用、API 调用失败或输出解析失败时，系统自动 fallback。

fallback 模式仍然可以：

- 读取上传数据；
- 使用本地规则和工具摘要；
- 生成 Manager Plan；
- 生成 Specialist Agent fallback 输出；
- 展示结果卡片；
- 生成行动项、报告和 Trace。

## 5. 安全边界

FinCopilot V2.2 仅用于财务整理、风险提醒和教育性支持。

系统不会提供投资、税务、法律、债务处置建议，不认定任何交易为欺诈，不执行真实付款或转账，也不承诺任何财务结果。
