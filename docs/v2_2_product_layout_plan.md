# FinCopilot V2.2 产品布局规划

## 1. 为什么要重构

V2.1 已经具备真实多 Agent 能力，但页面仍然较多，用户需要在多个页面之间跳转。V2.2 将主流程收敛到一个 Copilot 主界面，让用户可以在同一处上传数据、提出问题、查看核心回答、生成行动项并查看报告。

## 2. 新页面结构

- Copilot 主界面
- 分析详情
- 行动与报告
- 数据与设置

## 3. 主界面职责

Copilot 主界面承担：

- 数据快速上传；
- 数据状态展示；
- 推荐问题；
- 自然语言提问；
- Agent 回复；
- 建议行动；
- 补充问题；
- 报告入口；
- Manager Plan、Tool Results、Specialist Outputs 和 Trace 展开查看。

## 4. Agent API 策略

V2.2 默认尝试使用真实 Agent API。用户不需要选择是否启用 Agent。若 API Key 缺失、Key 无效、模型不可用、调用失败或输出解析失败，系统自动 fallback。

`ENABLE_AGENT_API=false` 只作为开发调试时的强制 fallback 开关，不作为普通用户界面选项。

## 5. 详情页职责

分析详情、行动与报告、数据与设置作为主界面的证据层和归档层：

- 分析详情：展示预算、发票、现金流、异常、目标和 Agent 工作台细节；
- 行动与报告：展示行动项状态、Multi-Agent 报告、Trace 和工作流报告；
- 数据与设置：展示数据上传与预览、API 状态、模型配置说明和安全边界。

## 6. V2.2 Step 2：主界面结果卡片化

V2.2 Step 2 将 `run_multi_agent_turn` 的结果转换为 `answer_presentation`，用于主界面展示。

展示结构包括：

- headline；
- summary；
- status badge；
- metric cards；
- risk cards；
- action cards；
- clarification cards；
- detail sections；
- report card。

这样用户不需要阅读完整长文本或 JSON，也能快速理解：

1. 当前结论是什么；
2. 风险在哪里；
3. 下一步该做什么；
4. 还需要补充什么；
5. 去哪里看详细结果。

## 7. V2.2 Step 3：主界面详细结果预览

Step 3 的目标是进一步减少用户在多个页面之间切换。

在 Step 2 中，主界面已经能展示结论、指标、风险和行动卡片。
Step 3 进一步让用户可以直接在主界面展开查看详细证据，包括现金流摘要、异常摘要、行动项、报告预览和 Agent 执行摘要。

详情页仍然保留，但定位变为完整表格、历史记录、下载归档和高级查看。

## 8. V2.2 Step 4：详情页重组

Step 4 将 V2.1 的多个页面正式合并为三个详情页：

1. 分析详情；
2. 行动与报告；
3. 数据与设置。

这样 Sidebar 中的四个入口与实际页面结构保持一致：

- Copilot 主界面用于上传、提问、查看核心结果；
- 分析详情用于查看完整财务分析和 Agent 执行轨迹；
- 行动与报告用于管理行动项、查看报告和 Trace；
- 数据与设置用于查看数据、字段说明、模型配置和安全边界。

## 9. V2.2 Step 5：新手引导与推荐任务

Step 5 的目标是降低新用户上手门槛。

Copilot 主界面会根据当前数据状态和最近一次 Agent 分析结果，动态推荐任务。

推荐任务包括：

- 检查未来 30 天现金流；
- 找出最可疑支出；
- 查看发票和付款优先级；
- 生成本周财务行动清单；
- 检查财务目标进度。

如果缺少数据，任务仍然展示，但会提示缺少哪些数据。

如果真实 Agent 不可用，系统会友好说明 fallback 模式仍可继续使用。

Step 6 完成 V2.2 的版本收尾，包括：

3. v2_2_checklist；
4. final demo plan；

完成后，项目可称为 FinCopilot V2.2 完整 Demo。
