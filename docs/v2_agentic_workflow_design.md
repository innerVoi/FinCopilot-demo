# FinCopilot v2 Agentic Workflow 设计文档

## 1. 设计目标

v2 的目标是从 v1 的 Dashboard + LLM Summary，升级为 Agentic CFO Copilot。

v2 不只是回答问题，而是帮助用户完成具体财务任务。

## 2. Agentic Workflow 定义

FinCopilot v2 的 agentic workflow 包括：

1. 理解用户任务；
2. 制定执行计划；
3. 检查数据完整性；
4. 主动追问缺失信息；
5. 调用确定性分析工具；
6. 汇总证据；
7. 生成行动清单；
8. 跟踪行动项状态；
9. 生成最终结果。

## 3. 工具调用原则

LLM 不直接计算财务指标，不直接判断异常。

确定性 Python 工具负责：

- 交易分类；
- 预算统计；
- 发票整理；
- 现金流风险计算；
- 财务目标进度分析；
- 规则异常检测；
- LOF 异常检测。

Agent 负责：

- 任务规划；
- 工具选择；
- 缺失信息识别；
- 追问澄清；
- 结果组织；
- 行动计划生成；
- 进展跟踪；
- 自然语言解释。

## 4. v2 计划实现的三个核心工作流

### 4.1 现金流安全检查 Workflow

用户问题：

“帮我判断未来 30 天现金流是否安全。”

执行步骤：

1. 检查交易、发票和目标数据是否存在；
2. 检查是否缺少真实账户余额、未来回款和未上传发票；
3. 调用预算分析工具；
4. 调用发票分析工具；
5. 调用现金流风险工具；
6. 调用异常检测工具；
7. 生成现金流风险结论；
8. 生成行动清单；
9. 提出必要澄清问题。

### 4.2 异常支出处理 Workflow

用户问题：

“帮我处理本月最可疑的支出。”

执行步骤：

1. 调用规则异常检测；
2. 调用 LOF 模型异常检测；
3. 合并异常结果；
4. 按风险等级和业务影响排序；
5. 对 Top 异常生成解释；
6. 生成核查清单；
7. 让用户标记处理状态。

### 4.3 财务目标行动计划 Workflow

用户问题：

“我想维持 30 天现金缓冲，应该怎么做？”

执行步骤：

1. 调用财务目标分析；
2. 调用预算分析；
3. 调用现金流分析；
4. 找出可关注支出类别；
5. 判断目标是否现实；
6. 追问缺失信息；
7. 生成行动计划；
8. 跟踪进展。

## 5. 低成本实现策略

v2 第一版不直接使用复杂 agent 框架，而是用规则化 workflow + 工具注册 + session_state 实现。

原因：

1. 稳定；
2. 低成本；
3. 易解释；
4. 易演示；
5. 不依赖复杂服务。

3. Agent 展示已调用工具；

## 8. V2 Step 2 实现状态

当前已实现 Agent Workspace 雏形：

1. 支持用户选择固定财务任务；
2. 支持展示 Agent 执行计划；
3. 支持数据完整性检查；
4. 支持生成澄清问题；
5. 支持展示计划调用的工具；

3. 行动项状态跟踪；
4. 多轮 Agent 记忆。

## 9. V2 Step 3 实现状态：工具注册与任务规划

当前已实现规则化工具注册、任务规划和工具执行轨迹：

1. `agent/tool_registry.py` 统一描述工具名称、输入、输出、类别、是否需要大模型和安全边界；
2. `agent/task_planner.py` 为现金流安全检查、异常支出处理和财务目标行动计划生成默认工具序列；
3. `agent/tool_executor.py` 在 Agent Workspace 中复用已有分析结果，必要时执行确定性 Python 工具；
4. Agent Workspace 可以展示可用工具表、结构化工具调用计划、planner 状态和每一步工具执行记录；
5. 大模型相关工具仍由用户主动触发，不在 Workspace 中自动批量执行；
6. 新增测试覆盖工具注册、任务规划和执行器。

Step 3 之后，Agentic Workflow 的核心骨架已经从“静态说明”升级为“可观察的任务计划与工具轨迹”。下一步将把工具结果转化为行动清单，并加入行动项状态跟踪。

## 10. V2 Step 4 实现状态：追问与上下文补全

当前已实现：

1. 任务相关澄清问题；
2. 业务上下文字段；
3. Agent session state；
4. 用户补充信息保存；
5. 澄清完成度计算；
6. 使用补充信息修正现金流摘要；
7. 使用补充信息补充异常分析摘要；
8. 使用补充信息补充目标分析摘要；
9. 在 Agent Workspace 中展示补充信息对分析结果的影响。

当前形成的闭环：

用户选择任务 → Agent 发现缺失信息 → Agent 提出澄清问题 → 用户补充信息 → Agent 保存上下文 → Agent 重新组织分析摘要 → Agent 展示补充信息影响。

2. 长期记忆；
